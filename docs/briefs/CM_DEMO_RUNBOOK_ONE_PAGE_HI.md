# CM Demo Runbook (एक पेज)
## उज्जैन कुंभ 2028 डिजिटल कमांड प्लेटफॉर्म

## 1) डेमो का उद्देश्य (2-स्क्रीन मॉडल)
- पब्लिक यूनिफाइड स्क्रीन: रजिस्ट्रेशन, OTP, QR पास, हेल्प/कम्प्लेंट और लुकअप।
- ऑप्स सेंटर: लाइव कतार, असाइन/रिजॉल्व, अर्जेंसी फीड, एस्केलेशन, एडमिन-ऑप्स टैब।

## 2) प्री-डेमो सेटअप (5 मिनट)
1. टर्मिनल में d:\Ultimate_QR\kumbh_system खोलें।
2. एक बार डिपेंडेंसी इंस्टॉल करें: npm install
3. सर्वर स्टार्ट करें (सुझावित): start-system.bat
4. वैकल्पिक स्टार्ट: set PORT=8090 && node server.js
5. हेल्थ चेक: http://127.0.0.1:8090/api/health

## 3) डेमो में खोलने वाले URL
- एंट्री हब: http://127.0.0.1:8090/
- पब्लिक यूनिफाइड: http://127.0.0.1:8090/ultimate-pro
- ऑप्स सेंटर: http://127.0.0.1:8090/ops (फॉलबैक: /control-room)
- एडमिन (लीगेसी, ऑप्शनल): http://127.0.0.1:8090/admin

## 4) डेमो लॉगिन क्रेडेंशियल्स
- Admin: admin / Admin@123
- Control Room: control / Control@123
- Police Officer: police / Police@123
- Queue Manager: queue / Queue@123

## 5) 12 मिनट का लाइव डेमो फ्लो
1. एंट्री हब (/) खोलकर 2-स्क्रीन आर्किटेक्चर समझाएं।
2. पब्लिक यूनिफाइड (/ultimate-pro) पर एक परिवार का रजिस्ट्रेशन करें।
3. OTP वेरीफिकेशन करें (मॉक मोड में OTP response में दिखेगा)।
4. QR पास जनरेट करके दिखाएं।
5. एक HELP या COMPLAINT टिकट बनाएं।
6. ऑप्स सेंटर (/ops) पर जाकर control से लॉगिन करें।
7. लाइव कतार में नया टिकट दिखाएं।
8. टिकट को अधिकारी को असाइन करें।
9. टिकट रिजॉल्व करें और CLOSED स्टेटस दिखाएं।
10. अर्जेंसी फीड, हीटमैप और एडमिन-ऑप्स कंट्रोल्स दिखाएं।

## 6) CM के लिए मुख्य संदेश
- नागरिक सेवा गति: शिकायत/हेल्प से कार्रवाई तक समय मिनटों में।
- फील्ड जवाबदेही: हर असाइन और रिजॉल्व क्रिया ट्रैक होती है।
- सुरक्षा प्राथमिकता: तेज शिकायत इनटेक और त्वरित फील्ड रिस्पॉन्स।
- स्केलेबल सिस्टम: उच्च भीड़ वाले कुंभ संचालन के लिए उपयुक्त।
- ऑडिट-रेडी गवर्नेंस: रोल-आधारित एक्सेस और लॉग ट्रेल।

## 7) कंटिंजेंसी प्लान (अगर कोई स्क्रीन न खुले)
1. एंट्री हब (/) से दुबारा नेविगेट करें।
2. /ops की जगह सीधे /control-room खोलें।
3. Admin या Control क्रेडेंशियल से दोबारा लॉगिन करें।
4. अगर ओपन टिकट न हो तो पब्लिक यूनिफाइड स्क्रीन से तुरंत नया टिकट बनाएं।

## 8) डेमो सफलता मानदंड
- रजिस्ट्रेशन OTP के साथ सफल हो।
- QR पास जनरेट हो।
- पब्लिक स्क्रीन से टिकट बने।
- ऑप्स सेंटर में टिकट असाइन और रिजॉल्व हो।
- डैशबोर्ड/क्यू में अंतिम स्टेटस CLOSED दिखे।

## 9) नवीनतम वैलिडेशन स्नैपशॉट
- पोर्ट 8090 पर फ्रेश इंस्टेंस वैलिडेट किया गया।
- /ops सही तरह से ऑप्स स्क्रीन पर मैप है।
- एंड-टू-एंड चेन पास: register -> OTP verify -> QR -> ticket -> assign -> resolve.
